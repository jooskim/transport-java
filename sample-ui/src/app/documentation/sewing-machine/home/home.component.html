Sewing Machine is an intent compiler

Traditionally compilers translate a language into machine code, a transpiler translates code from one language to another. What is all this 'intent' compiler then?

The compiler works by taking user intentions and compiling them into applications, services, components and operations. The sewing machine can automatically build the lions share of your application's data and infrastructure architecture. It does this by employing some strict architectural principals and opinionated tooling, but in a universal and technology independent way.

What are you talking about?

Let's take a typical example of a what an engineer needs to do when asked to build a new product that requires customer experience/interactivity:

  1. Run the gauntlet of researching the latest technology shift in UI technologies
  2. Ensure choices are compatible with existing stack & existing product(s)
  3. Map out backend APIs required to meet UX demands
  4. Negotiate API contract specifications with backend (if at all)
  5. Design middleware to support APISs that cannot be managed or consumed directly
  6. Map out UI business logic required to support APIs
  7. Map out middleware business logic required to support orchestration / persistence
  8. Map out UI & middleware architecture and technology testing (e2e & unit)
  9. Map out non RESTful standards into middleware (pub-sub/gRPC/graphQL etc)
  10. Build out UI and middleware foundations
  11. Build out UI and middleware business logic against shifting API contracts.. and maintain as it shifts.

Once all that is done, then you can start to build a new product. This process can take months and of engineers time.

The sewing machine will do all of the above for you in seconds.

Not only will it build out every single line of business logic, models and interfaces for your APIs, it will also build your 'ready-to-modify' UI components.
