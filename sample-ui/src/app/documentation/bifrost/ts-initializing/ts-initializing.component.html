Initializing the Bus

The Bifröst operates outside of any vendor UI technology framework. It's completely decoupled, allowing it to be used by any framework. Because it's independent, it needs to be initialized before any other frameworks have booted or started e.g. Angular.

There are a couple of ways to do this, the first is to create an initialization script that is auto-loaded into the <head/> of the DOM (and would therefore boot before the UI framework does).

The quickest way to initialize the Bifröst is to boot it in your app.component.ts class (for Angular). This should be done outside your actual AppComponent class definition, to ensure the Bifröst is booted before your application is. Your app.component.ts file would end up looking something like this: 


import { Component } from '@angular/core';
import { BusUtil } from '@vmw/bifrost/util/bus.util';

// create bus instance with debug level logging.
BusUtil.bootBusWithOptions(LogLevel.Debug, false);

@Component({
    selector: 'app-root',
    templateUrl: './app.component.html',
    styleUrls: ['./app.component.scss']
})
export class AppComponent {

}

The Bifröst is now active and ready to be be used. If you want to just boot the bus with default options, you can use BusUtil.bootBus(). You can find out about some of the other methods available by looking at the BusUtil API, you can find the code here: https://gitlab.eng.vmware.com/bifrost/typescript/blob/master/src/util/bus.util.ts
