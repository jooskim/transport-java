Sewing Machine Quickstart

The quickest way to get started with the Sewing Machine is to use App-o-matic to build an entire, ready to run application, that uses a sample API contract. This sample API runs on this site, so you can try it all out yourself for real.

Step 1: Grab the SewingMachine toolchain

Don't worry, you don't need to mount anything or set up a bunch of environmental variables, all you need to do is pull down our simple build script that will handle everything for you.

Create a new folder called 'fabric-tools'

> mkdir ~/fabric-tools && cd ~/fabric-tools

Grab the Sewing Machine build script

> wget http://appfabric.vmware.com/assets/build-sewingmachine.sh -O build-sewingmachine.sh

Make the script executable

> chmod +x build-sewingmachine.sh

Step 2: Build all the things

> ./build-sewingmachine.sh latest

You can use latest or the version (v2.xxx)

You will see docker complaining about not finding local images of the sewing-machine and will then proceed to download the latest Docker image that actually compiles all our of tooling. Your console will output something like the below (along with a bunch of Docker output)

Building apigen for darwin amd64...
Building servgen2 for darwin amd64...
Building linty for darwin amd64...
Building modelgen for darwin amd64...
Building appgen for darwin amd64...
Building appgen2 for darwin amd64...
Building specgen for darwin amd64...
Building appomatic for darwin amd64...
If you're on a mac, you will see darwin amd64, if you're running another OS with a different proc type, you will see something different.

Step 3: Download our sample API Specification

In order to create a useful application, you need some APIs to call right? You can use our sample OpenAPI specification as a sample to get going. You can download it in the same way you downloaded the build script. 

> wget http://appfabric.vmware.com/assets/fabric-sample-services-api-doc.yaml -O fabric-sample-services-api-doc.yaml

Step 4: Build your sample app!

Now you have the tooling and a valid API contract for a 'service'. Now you're ready to build. We're going to call our app 'sample-app' 

> ./appomatic --new -dvt --alias=@sample-app --add-YAML=fabric-sample-services-api-doc.yaml --autogen ~/sample-app
We have added the --dvt flag here, this will print out verbose, debug and trace level output, so you can see what is going on. There should be a number of messages generated by the sewing machine.

Step 5: Run your sample App!

Once your sample app has been built, you can run it! 

> cd ~/sample-app && npm start

Your brand new, ready to go (but empty) app should boot and you should be able to load it up in a browser on http://localhost:4200.
