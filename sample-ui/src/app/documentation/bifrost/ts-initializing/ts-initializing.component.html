Initializing the Bus

The Bifr√∂st operates outside of any vendor UI technology framework. It's completely decoupled, allowing it to be used by any framework. Because it's independent, it needs to be initialized before any other frameworks have booted or started e.g. Angular.

There are a couple of ways to do this, the first is to create an initialization script that is auto-loaded into the <head/> of the DOM and would therefore boot before the UI framework does. This init/boot script serves as a great spot to set up and configure all your fabric needs.

Booting via init script

The simplest and quickest way to initialize the bus is in this init script, you just need a single line of code.

Simplest

import { BusUtil } from '@vmw/bifrost/util/bus.util';

// initializing the event bus.
BusUtil.bootBus();

With Options

import { BusUtil } from '@vmw/bifrost/util/bus.util';

// initializing the event bus, with debug logging enabled, and the boot message is not disabled.
BusUtil.bootBusWithOptions(LogLevel.Debug, false);

The Bifr√∂st is now active and ready to be be used. It operates as a singleton inside your application. You can access the bus in a couple of ways.

  1. Using the built in Abstractions ‚ú®(recommended)
  2. Accessing the bus via the window object, via window.AppEventBus

Using the provided Abstractions is the easiest and simplest way to integrate the bus. It's the same approach all Fabric code uses, everywhere.

The second option is useful for non-TypeScript applications. Applications using vanilla JavaScript would use this method.

You can find out about some of the other methods available by looking at the BusUtil API, you can find the code here: https://gitlab.eng.vmware.com/bifrost/typescript/blob/master/src/util/bus.util.ts
